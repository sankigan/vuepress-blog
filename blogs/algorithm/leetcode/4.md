> [寻找两个正序数组的中位数](https://leetcode.cn/problems/median-of-two-sorted-arrays/description/)

## 题目描述

给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。请你找出并返回这两个正序数组的**中位数**。

算法的时间复杂度应该为 `O(log (m+n))` 。

**示例**

```
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组 = [1,2,3] ，中位数 2

输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5
```

**提示**

- `nums1.length == m`
- `nums2.length == n`
- `0 <= m <= 1000`
- `0 <= n <= 1000`
- `1 <= m + n <= 2000`
- `-10^6 <= nums1[i], nums2[i] <= 10^6`

## 解答

### 解法一：归并双指针（`O(m + n)`）

#### 关键思路

最直观的思路：既然两个数组都已经有序，那就用归并排序的「合并」步骤把它们拼成一个新有序数组，再按下标取中位数。

1. 用双指针 `p1`、`p2` 分别指向 `nums1`、`nums2` 的当前最小未取元素；
2. 每轮比较两指针所指的值，把较小者写入合并数组的 `p1 + p2` 位置，对应指针前进一位；
3. 任一数组耗尽后，把另一数组的剩余元素直接拷贝到末尾；
4. 总长度 `len = m + n` 的中位数有两种情况：
    - 奇数：取下标 `(len - 1) / 2` 处的元素；
    - 偶数：取下标 `len / 2 - 1` 和 `len / 2` 两个元素的均值。

技巧：令 `mid = (len - 1) / 2`，取 `low = floor(mid)` 与 `high = ceil(mid)`，返回 `(arr[low] + arr[high]) / 2`。这样奇偶可以统一表达——奇数时 `low === high`，相当于该元素自身；偶数时 `low` 和 `high` 是中间两个下标。

#### 复杂度

- 时间：`O(m + n)`，每个元素入合并数组一次。
- 空间：`O(m + n)`，合并数组。

> 该解法不满足题目进阶的 `O(log(m + n))` 要求，但思路清晰、易写易调试，适合作为兜底。

::: details Code
```ts
function findMedianSortedArrays(nums1: number[], nums2: number[]): number {
  const m = nums1.length;
  const n = nums2.length;
  const arr: number[] = new Array(m + n);
  let p1 = 0;
  let p2 = 0;

  while (p1 < m || p2 < n) {
    // 一侧耗尽，另一侧整段补完后退出
    if (p1 === m) {
      while (p2 < n) arr[p1 + p2] = nums2[p2++];
      break;
    }
    if (p2 === n) {
      while (p1 < m) arr[p1 + p2] = nums1[p1++];
      break;
    }

    // 取两指针所指较小者写入合并位置
    if (nums1[p1] < nums2[p2]) {
      arr[p1 + p2] = nums1[p1++];
    } else {
      arr[p1 + p2] = nums2[p2++];
    }
  }

  // 奇偶统一: low === high 时退化为单元素自身
  const mid = (arr.length - 1) / 2;
  const low = Math.floor(mid);
  const high = Math.ceil(mid);
  return (arr[low] + arr[high]) / 2;
}
```

### 解法二：划分二分（`O(log min(m, n))`）

#### 关键思路

把求中位数转化为一个更一般的问题：**在两个有序数组中找合并后第 `k` 小的元素**。中位数就是 `k = (m + n + 1) / 2`（奇数）或第 `k` 与第 `k + 1` 小的均值（偶数）。

剩下的问题是：怎么快速找到「合并后的第 `k` 小」而不真的合并？关键在于**每轮排除一批一定不是答案的元素**，让 `k` 不断减小。

##### 为什么每次比较第 `k / 2` 个元素？

想象一下：如果我们在 `nums1` 中取前 `a` 个、在 `nums2` 中取前 `b` 个，要求合起来恰好就是「最小的 `k` 个」，那必须 `a + b = k`。

我们一次只想稳妥地排除一批元素，但又不知道答案最终在哪一侧偏多，最公平的做法就是**两边各试 `k / 2` 个**——这是一个对称的切分：

- 如果取得**太少**（比如各取 1 个），一轮只能排掉 1 个元素，复杂度退化为 `O(k)`，跟归并没区别；
- 如果取得**太多**（比如某一侧直接取 `k` 个），万一答案其实集中在另一侧，就可能**误删第 `k` 小本身**，因为你试探的位置已经越过了真正的答案；
- **各取 `k / 2`** 是唯一兼顾「一次排掉够多」和「绝对不会误删答案」的切法——下一节会证明这一点。

形式化地说：比较 `nums1[k/2 - 1]` 与 `nums2[k/2 - 1]`（数组从 0 计数，所以 `-1`）这两个候选位置。

##### 为什么较小那一侧的前 `k / 2` 个可以直接排除？

直接讲抽象式子容易绕，先**代具体数字**走一遍，再回到通式就清楚了。

假设 `k = 6`，两数组前 3 个元素长这样（用 `A` 代 `nums1`，`B` 代 `nums2`）：

```
A: [ 1,  3,  5 | ... ]   ← A[2] = 5  (较小候选)
B: [ 2,  4,  9 | ... ]   ← B[2] = 9  (较大候选)
```

两边各取第 3 个比较：`A[2] = 5 < B[2] = 9`。问：**`A[0..2]` 这 3 个元素能不能是第 6 小？**

答案是：**最多排到第 5 小，不可能是第 6 小。** 数一下小于等于 `A[2] = 5` 的元素最多有几个：

- 在 `A` 里：`1, 3, 5`，共 **3 个**（`A[2]` 自己和前面 2 个）；
- 在 `B` 里：因为 `B[2] = 9 > 5`，而 `B` 升序，`B[2]` 后面的更大，所以能 `≤ 5` 的最多就是 `B[0], B[1]` 这 **2 个**（实际是不是 ≤ 5 不重要，**最多就 2 个**）。

加起来：`3 + 2 = 5`。所以 `A[2]` 在合并数组里排名**最多第 5**，连第 6 小都够不上，更别说 `A[0]`、`A[1]` 这俩比它还小的了。**整段 `A[0..2]` 可以安全删掉。**

回到通式，把 `3` 换成 `k / 2`：

- `A` 那侧贡献 `k / 2` 个（候选自己 + 前面那些）；
- `B` 那侧因为有 `B[k/2 - 1] ≥ A[k/2 - 1]` 当"屏障"，能 `≤` 较小候选的**最多 `k / 2 - 1` 个**；
- 合计排名上界 = `k / 2 + (k / 2 - 1) = k - 1` < `k` ✅

排除掉这 `k / 2` 个后，问题变成「在 `A` 剩下部分与 `B` 中找第 `k - k/2` 小」，**`k` 至少减半**，递归 `O(log k) = O(log(m + n))`。

> **反过来想**：为什么不能排 `B` 那一侧？还是上面的例子，`k = 6`，把 `B` 改成 `[2, 4, 6, 7, 8]`：`A[2] = 5 < B[2] = 6`，但合并后是 `[1, 2, 3, 4, 5, 6, 7, 8]`，第 6 小恰好是 `B[2] = 6` 本身。所以**较大那一侧的前 `k / 2` 个不能动**——里面可能藏着答案。

##### 算法骨架

1. 把"求中位数"包装成"求第 `k` 小"：奇数取一次，偶数取两次再求均值；
2. 内层 `getKth(i, j, k)` 表示从 `nums1[i..]` 与 `nums2[j..]` 中找第 `k` 小：
    - **边界 1**：若某一侧已用完（`i >= m` 或 `j >= n`），直接在另一侧定位第 `k` 个；
    - **边界 2**：若 `k === 1`，返回两侧首元素较小者；
    - **常规**：两侧分别取第 `k / 2` 个候选 `mid1 = i + k/2 - 1`、`mid2 = j + k/2 - 1`（越界时夹到数组末尾，避免读到 `undefined`），比较后淘汰较小候选所在那段，`k` 减去淘汰掉的元素数，递归。

每轮 `k` 至少减半，因此对数复杂度。

#### 复杂度

- 时间：`O(log(m + n))`，更精确地是 `O(log min(m, n))`。
- 空间：`O(1)`（迭代写法）或 `O(log(m + n))`（递归调用栈）。

::: details Code
```ts
function findMedianSortedArrays(nums1: number[], nums2: number[]): number {
  const m = nums1.length;
  const n = nums2.length;
  const total = m + n;

  // 在 nums1[i..] 与 nums2[j..] 中找第 k 小 (k 从 1 计数)
  const getKth = (i: number, j: number, k: number): number => {
    // 边界 1: 任一侧用完, 直接在另一侧定位
    if (i >= m) return nums2[j + k - 1];
    if (j >= n) return nums1[i + k - 1];
    // 边界 2: 取首元素较小者
    if (k === 1) return Math.min(nums1[i], nums2[j]);

    // 两侧分别取第 k/2 个候选 (夹到数组末尾防越界)
    const half = Math.floor(k / 2);
    const mid1 = Math.min(i + half, m) - 1;
    const mid2 = Math.min(j + half, n) - 1;

    // 淘汰较小候选所在那段, k 同步减去被淘汰的元素数
    if (nums1[mid1] <= nums2[mid2]) {
      return getKth(mid1 + 1, j, k - (mid1 - i + 1));
    }
    return getKth(i, mid2 + 1, k - (mid2 - j + 1));
  };

  // 奇数: 第 (total + 1) / 2 小; 偶数: 第 total/2 与 total/2 + 1 小的均值
  if (total % 2 === 1) {
    return getKth(0, 0, (total + 1) >> 1);
  }
  return (getKth(0, 0, total >> 1) + getKth(0, 0, (total >> 1) + 1)) / 2;
}
```
